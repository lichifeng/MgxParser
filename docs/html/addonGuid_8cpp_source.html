15 /**
16  * \brief Guid is presented in HD/DE records, but not in older versions. Guid can be used to identify different views of same game.
17  * \details This function will combine some stable info among different views of same game and do a md5 hashing to get a uniform code
18  * for them. It is used mainly in records of older versions which have no native guid embeded in record file.
19  * MgxParser uses following data to generate guid of a game (* means optional):
20  * | versionStr
21  * | saveVersion
22  * | logVersion
23  * | scenarioVersion
24  * | mapDataPtr
25  * | mapSize
26  * | popLimit
27  * | gameSpeed
28  * | PlayerName + civID + index + slot + colorID + teamID by slot sequence
29  * | First 20 bytes of first MOVE action data in body
30  * | Happen time of first MOVE action
31  * Note: Duration is normally different among records of same game, 'cuz player
32  * quit at different time. That's why duration cannot be a factor of this
33  * and only the very early action data can be used.
34  *
35  * \return string Guid of this record
36  */
